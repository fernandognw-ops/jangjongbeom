# 출고 병합 제거 - 실행 보고

## 실행 순서

### 1. Supabase SQL Editor에서 마이그레이션 실행

**파일**: `scripts/migrate_outbound_no_merge.sql`

1. [Supabase 대시보드](https://supabase.com/dashboard) → 프로젝트 선택
2. **SQL Editor** → **New query**
3. `scripts/migrate_outbound_no_merge.sql` 내용 전체 복사 후 붙여넣기
4. **Run** 클릭
5. `출고 unique 제약 제거 완료. 1 row = 1 트랜잭션.` 메시지 확인

---

### 2. 기존 병합된 outbound 데이터 정리

- **옵션 A**: Excel 재업로드 시 commit 로직이 해당 날짜 범위 outbound를 자동 삭제 후 insert → 별도 정리 불필요
- **옵션 B**: 전체 초기화 후 재업로드 시 → Supabase SQL Editor에서 `TRUNCATE inventory_outbound;` 실행

---

### 3. 동일 Excel 웹 업로드

1. `npm run dev` 실행 (또는 배포 URL 접속)
2. 대시보드 → **Excel 업로드**
3. 생산수불현황 Excel 파일 선택

---

### 4. 업로드 전 검증 수치 확인

| 항목 | 기대값 | 실제값 |
|------|--------|--------|
| rawdata | 480 | |
| inbound | 172 | |
| outbound | 2965 | |
| stock | 414 | |

수치가 일치하면 **DB 반영** 클릭.

---

### 5. 반영 후 확인

- `node scripts/run_outbound_migration.mjs --verify` 실행
- 또는 `/api/inventory/quick` 호출 결과 확인

---
