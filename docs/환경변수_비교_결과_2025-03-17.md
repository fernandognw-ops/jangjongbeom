## 3. env 변경 후 dev 서버 재시작 필요 여부

**필수입니다.**

- `NEXT_PUBLIC_*` 변수는 **빌드/시작 시점**에 로드됨
- `.env.local` 변경 후에는 **반드시 dev 서버 재시작** 필요
- `next dev`는 시작 시 `.env.local`을 읽고, 이후 변경은 반영되지 않음

## 4. localhost 0건 반환을 위한 조치

**현재 상황**: `.env.local`과 Vercel이 동일한 프로젝트 `slnmhblsxzjgmaqbfbwa`를 사용함에도 불구하고:
- **로컬**: 326건 반환 (inventory_stock_snapshot에 데이터 있음)
- **Vercel**: 0건 반환 (no_snapshot, 테이블 비어 있음)

이는 다음 중 하나를 의미합니다:
1. **Vercel 대시보드**에 설정된 환경변수가 `vercel env pull` 결과와 다를 수 있음
2. **Supabase 프로젝트가 2개** 존재할 가능성 (0건 프로젝트 vs 326건 프로젝트)

### 권장 조치

1. **[Vercel 대시보드](https://vercel.com)** → 프로젝트 → **Settings** → **Environment Variables**
   - `NEXT_PUBLIC_SUPABASE_URL` 값 확인 (0건을 반환하는 프로젝트의 URL)
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` 확인

2. **0건을 반환하는 Supabase 프로젝트**의 URL에서 project ref 추출
   - 예: `https://XXXXX.supabase.co` → ref는 `XXXXX`

3. **`.env.local`을 해당 ref로 수정** 후 dev 서버 재시작
   ```bash
   # 3007 포트 프로세스 종료 후
   npm run dev
   ```

4. **동기화 확인**: `npx vercel env pull .env.vercel.production --environment=production` 실행 후 `.env.local`과 비교

## 5. Vercel env pull 명령 (참고)

```bash
npx vercel env pull .env.vercel.production --environment=production
```

생성된 `.env.vercel.production`과 `.env.local`을 비교하여 동기화할 수 있습니다.
