# 첫 업로드 검증 가이드

웹 업로드로 첫 데이터 반영 후 검증 절차

---

## 1. 테스트 파일 1개 업로드

1. 대시보드 접속
2. **생산수불현황 업로드** 영역에 Excel 파일 드래그 또는 선택
3. **업로드 전 검증** 확인:
   - rawdata 건수
   - 입고 건수
   - 출고 건수
   - 재고 건수
   - 재고 총 금액
   - 일반 / 쿠팡 분포
   - snapshot_date
4. 이상 없으면 **DB 반영** 클릭

---

## 2. 업로드 후 검증

### 2-1. Supabase SQL 검증

`scripts/verify_after_first_upload.sql` 실행

| 항목 | 기대 결과 |
|------|----------|
| inventory_products | 0보다 큼 |
| inventory_inbound | 0 이상 |
| inventory_outbound | 0 이상 |
| inventory_stock_snapshot | 0보다 큼 |
| duplicate_count | 0 (중복 없음) |
| dest_warehouse | 일반, 쿠팡 분포 |
| total_stock_value | 0보다 큼 |
| snapshot_date | 최신 날짜 존재 |

### 2-2. API 검증

```bash
npm run verify-reset -- http://localhost:3007
```

- **초기화 상태(0건)**: exit 0, "초기화 정상 완료"
- **데이터 반영 후**: exit 1 (정상, 데이터 있음)
- `/api/inventory/quick` 응답: productCount > 0, totalValue > 0

### 2-3. 웹 화면 확인

- **KPI**: 품목 수, 재고 금액, 수량, SKU 정상 표시
- **그래프**: 카테고리별 출고 데이터 표시
- **재고 분포**: 일반/쿠팡 분포 정상
