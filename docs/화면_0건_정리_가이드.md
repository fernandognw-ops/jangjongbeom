# 화면 0건 정리 가이드

Supabase 단일 출처 모드에서 이전 데이터가 화면에 남지 않도록 정리한 내용입니다.

---

## 1. inventory_sync 2건 화면 복원 사용 여부

### 확인 결과: **일반적으로 사용되지 않음**

| sync_code | 사용 조건 |
|-----------|------------|
| MAIN | `getDefaultWorkspaceId()` → "MAIN" 반환 시 `fetchDefaultWorkspace()` 호출. **현재 inventory_sync에 MAIN 행 없음** → 복원 안 됨 |
| B9T3EQSMNHVK, TQKAXFDNHHQK | `getStoredSyncCode()`가 해당 코드를 반환할 때만 사용. localStorage `inventory-sync-code`에 해당 값이 있어야 함 |

**결론**: 기본 워크스페이스(MAIN) 행이 없어, 대부분의 경우 inventory_sync는 화면 복원에 사용되지 않습니다.  
다만 `inventory-sync-code`에 B9T3EQSMNHVK 또는 TQKAXFDNHHQK가 저장된 기기에서는 복원에 사용될 수 있습니다.

## 2. inventory_sync 안전 삭제 방법

### 사전 확인
```bash
npm run check-inventory-sync
```

### 삭제 절차
1. Supabase 대시보드 → SQL Editor
2. `scripts/truncate_inventory_sync.sql` 내용 실행
3. 1) 확인 쿼리로 현재 데이터 검토
4. 2) `TRUNCATE inventory_sync;` 실행
5. 3) 삭제 결과 확인

## 3. localStorage inventory-* 키와 화면 표시 관계

| 조건 | 화면 표시 |
|------|-----------|
| useSupabaseInventory = **true** (Supabase 사용) | API 응답만 사용, **localStorage 영향 없음** |
| useSupabaseInventory = **false** (로컬 모드) | baseStock, transactions, products 등 **localStorage 기반** 표시 |

### Supabase 단일 출처 모드(SUPABASE_SINGLE_SOURCE=true) 적용 시
- **empty_data**: localStorage `inventory-*` 삭제 후 0건 표시
- **fetch_error**: inventory_sync 복원 안 함, localStorage 삭제 후 0건 표시
- **로컬 모드로 전환**: 복원 안 함, localStorage 삭제 후 0건 표시

---

## 4. 배포 화면 새로고침/초기화 절차

### 4.1 강력 새로고침 (권장)
- **PC**: Ctrl+Shift+R (Windows) / Cmd+Shift+R (Mac)
- **모바일**: 브라우저 설정 → 캐시/데이터 삭제 후 URL 재접속

### 4.2 localStorage 수동 삭제
1. F12 → 개발자 도구 → Application 탭
2. Local Storage → 배포 URL 선택
3. `inventory-`로 시작하는 키 삭제
4. 페이지 새로고침

### 4.3 콘솔로 일괄 삭제
```javascript
Object.keys(localStorage)
  .filter(k => k.startsWith('inventory-'))
  .forEach(k => localStorage.removeItem(k));
location.reload();
```

## 5. 로컬 모드 복원 경로 임시 비활성화

`src/context/InventoryContext.tsx`에 `SUPABASE_SINGLE_SOURCE = true` 설정됨.

### 비활성화된 동작
- API 실패 시 inventory_sync에서 복원하지 않음
- "로컬 모드로 전환" 버튼 숨김
- empty_data / fetch_error 시 localStorage 삭제 후 0건 표시

### 복원 방법 (로컬 모드 다시 사용할 때)
`SUPABASE_SINGLE_SOURCE`를 `false`로 변경 후 재배포.
