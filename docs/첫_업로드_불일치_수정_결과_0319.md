# 첫 업로드 불일치 수정 결과 (2026-03-19)

## 1. rawdata 누락 원인 분석 (480 vs 355)

### 원인
- **웹 파서**: rawdata 시트를 파싱하지 않음. `currentProductCodes` = 입고+출고+재고 품목코드 union = 355건
- **inventory_products**: 입고/출고/재고에 등장하는 품목코드만 FK 보완으로 추가 → 355건
- **기대**: rawdata 시트 480건 → inventory_products 480건

### 수정
- `parseRawdataSheet()` 추가 (excelParser)
- rawdata 시트 검색: rawdata, 제품현황_일반, 제품현황_상세, 제품현황, 품절관리
- rawdata 있으면 전체 upsert → inventory_products 480건
- 검증 `rawdataCount` = rawdata.length (있으면) 또는 currentProductCodes.length (없으면)

---

## 2. 입고 누락 원인 분석 (172 vs 168)

### 원인
- **적재 로직**: (product_code, inbound_date)당 1행으로 병합, 수량 합산
- **DB unique**: (product_code, inbound_date) → 1행만 허용
- **결과**: 172건 → 168건 (동일 product+date 4건 병합)

### 수정
- unique 제약 변경: (product_code, inbound_date, dest_warehouse)
- 병합 제거: 검증 172건 = 적재 172건
- 마이그레이션: `scripts/migrate_inbound_outbound_unique_for_row_count.sql`

---

## 3. 출고 누락 원인 분석 (2965 vs 2798)

### 원인
- **적재 로직**: (product_code, outbound_date, sales_channel)로 병합
- **DB unique**: (product_code, outbound_date, sales_channel)
- **결과**: 2965건 → 2798건 (동일 product+date+channel 167건 병합)

### 수정
- unique 제약 변경: (product_code, outbound_date, dest_warehouse)
- 병합 기준 변경: (product_code, outbound_date, dest_warehouse)로 합산
- 검증 outboundCount = 병합 후 건수 (적재 예정 건수와 동일)

## 4. /api/inventory/quick no_snapshot 해결

### 원인
- order("snapshot_date", { ascending: false }) 쿼리가 0건 반환 (로컬 env/캐시 이슈 가능)
- Supabase 프로젝트 ref: slnmhblsxzjgmaqbfbwa

### 수정
- order 없이 limit(500) fallback 추가
- fallback에서 snapshot_date 최대값 계산 후 해당 날짜로 재조회
- 개발 모드에서 query1 로그 출력
