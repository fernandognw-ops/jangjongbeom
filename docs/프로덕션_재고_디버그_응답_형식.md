# `/api/inventory/quick?debug=1` — 프로덕션 강제 디버깅

배포 후 브라우저 또는 `curl`로 호출:

```text
GET https://<배포도메인>/api/inventory/quick?debug=1
```

## 응답에 포함되는 필드 (debug=1일 때)

### 루트 레벨

| 필드 | 설명 |
|------|------|
| `channelTotals` | 채널별 수량 합 (`쿠팡` / `일반` 키) |
| `snapshot_date` | 집계에 사용한 `snapshot_date` (YYYY-MM-DD) |
| `row_count` | 해당 일자 스냅샷 행 수 |
| `debug_used_channel_source_counts` | `sales_channel_used`, `dest_warehouse_fallback_used`, `empty_source`(선택) |

### `_debug` 객체

| 필드 | 설명 |
|------|------|
| `query1`, `query2` | Supabase 조회 메타 |
| `debug_*` | 집계기에서 나온 상세 (아래 `debug_aggregate`와 동일) |
| `ui_data_path_note` | **화면이 실제로 쓰는 데이터 경로** 안내 |

### 집계 상세 (`debug_aggregate` 내용이 `_debug`에 병합됨)

- `debug_sample_rows`: 집계 순서 기준 **앞 20건** (product_code, storage_center, dest_warehouse, sales_channel, resolution_source, chosen_channel, quantity)
- `debug_chosen_channel_counts`: `coupang` / `general` 행 건수
- `debug_rows_sales_nonempty_and_chosen_coupang`: `sales_channel`이 비어 있지 않고 `chosen_channel === 쿠팡`인 행 최대 20건
- `debug_anomaly_sales_normalizes_general_but_chosen_coupang`: **버그 후보** — `sales_channel` 정규화가 일반인데 `chosen_channel`이 쿠팡인 행 (비어 있으면 정상)

## 화면(대시보드)이 참조하는 API

### 상단·하단 채널 수치 (`DashboardBoxHero`)

- `channelTotals` ← `InventoryContext`의 `channelTotals`
- `refresh()` 흐름:
  1. **`/api/inventory/quick`** 먼저 호출 → `setChannelTotals(data.channelTotals)`
  2. **백그라운드**로 **`/api/inventory/snapshot`** (lite 아님) 호출 → 성공 시 `channelTotals`를 **다시 덮어씀**

따라서 **최종 화면 숫자**는 `quick`만이 아니라 `snapshot`의 `channelTotals`와 같아야 함. 둘이 다르면 `snapshot` 응답에 `?debug`를 붙이지는 않았으므로, 브라우저 Network에서 `snapshot` JSON의 `channelTotals`와 `quick?debug=1`의 `channelTotals`를 직접 비교할 것.

### 캐시

- `fetch`에 `cache: "no-store"` 및 `cacheBust=_t=` 사용.
