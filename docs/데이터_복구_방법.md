# 데이터 복구 방법 (데이터 사라짐 시)

현재 **재고 금액 0원**, **no_snapshot** 오류가 나는 경우 아래 순서로 복구하세요.

---

## 방법 1: Supabase 일일 백업 복원 (가장 먼저 시도)

Supabase는 **무료 플랜**에서도 **일일 백업**을 자동 생성합니다.

### 절차

1. [Supabase 대시보드](https://supabase.com/dashboard) 로그인
2. 프로젝트 선택: **slnmhblsxzjgmaqbfbwa**
3. **Database** → **Backups** 이동
4. **Scheduled Backups**에서 **삭제 전 시점** 백업 선택
   - 예: 오늘 아침 또는 어제 백업
5. **Restore** 클릭 후 확인

> ⚠️ 복원 중에는 프로젝트가 일시적으로 접근 불가합니다.  
> 복원 후 대시보드 새로고침하면 데이터가 다시 보입니다.

---

## 방법 2: 생산수불현황 엑셀로 재적재

백업 복원이 어렵거나, 백업 시점이 너무 오래된 경우:

### 2-1. bulk-upload (웹 API 경유)

1. **생산수불현황 엑셀 파일** 준비  
   - 예: `(수정양식)26년 0318_생산수불현황.xlsx` 또는 수불 마감 자료
2. 터미널에서:

```powershell
cd c:\Users\pc\inventory-system
npm run dev
```

3. **새 터미널**에서:

```powershell
npm run bulk-upload "엑셀파일경로"
# 예: npm run bulk-upload "C:\Users\pc\Desktop\26년_0318_생산수불현황.xlsx"
```

4. `http://localhost:3007` 접속 후 새로고침

### 2-2. sync-excel (Python 직접 Supabase 적재)

```powershell
python scripts/integrated_sync.py "엑셀파일경로"
```

> `bulk-upload`는 dev 서버가 떠 있어야 하고,  
> `sync-excel`은 dev 서버 없이 바로 Supabase에 적재합니다.

---

## 방법 3: Supabase 프로젝트 불일치 확인

**증상**: bulk-upload는 성공하는데 대시보드에 0원만 표시

**원인**: 로컬 `.env.local`의 Supabase와 Vercel 배포 URL의 Supabase가 다를 수 있음

**확인**:
- Vercel → Settings → Environment Variables  
- `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`가 `.env.local`과 동일한지 확인

---

## 권장 순서

1. **Supabase Backups**에서 삭제 전 시점 백업 복원 시도
2. 복원 불가 시 → **생산수불현황 엑셀**로 bulk-upload 또는 sync-excel 실행
3. 여전히 0원이면 → Supabase 프로젝트 ID·환경변수 일치 여부 확인
