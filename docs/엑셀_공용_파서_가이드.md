# 엑셀 공용 파서 가이드

로컬(integrated_sync.py)과 웹(productionSheetParser)이 **동일한 규칙**으로 엑셀을 파싱합니다.

---

## 1. 공용 구조

| 파일 | 용도 |
|------|------|
| `common/rules.py` | Python 규칙 상수 (헤더 행, 동의어, 제외 키워드) |
| `common/parser.py` | Python 파싱 로직 |
| `common/classifier.py` | Python warehouse_group 분류 |
| `src/lib/excelParser/rules.ts` | TypeScript 규칙 (동일) |
| `src/lib/excelParser/parser.ts` | TypeScript 파싱 로직 (동일) |
| `src/lib/excelParser/classifier.ts` | TypeScript 분류 (동일) |

---

## 2. 시트별 규칙 (고정)

### 입고 (Inbound)
- **헤더 행**: 2행 (0-indexed = 1)
- **데이터 시작**: 3행 (0-indexed = 2)

| 엑셀 컬럼 | 매핑 |
|----------|------|
| 품번, 품목코드 | product_code |
| 상품명, 품목명, 제품명 | product_name |
| 입고 수량, 수량 | quantity |
| 입고 센터, 입고처, 창고 | inbound_center → dest_warehouse |
| 입고일자, 입고일 | inbound_date |

### 출고 (Outbound)
- **헤더 행**: 2행
- **데이터 시작**: 3행

| 엑셀 컬럼 | 매핑 |
|----------|------|
| 품번, 품목코드 | product_code |
| 상품명, 품목명, 제품명 | product_name |
| 출고 수량, 수량 | quantity |
| 출고 센터, 출고처 | outbound_center → dest_warehouse |
| 출고일자, 출고일 | outbound_date |
| 매출구분, 판매처 | sales_channel (없으면 warehouse_group 기반) |

### 재고 (Stock)
- **헤더 행**: 2행
- **데이터 시작**: 3행

| 엑셀 컬럼 | 매핑 |
|----------|------|
| 품번, 품목코드 | product_code |
| 상품명, 품목명, 제품명 | product_name |
| 재고 수량, 수량 | quantity |
| 보관 센터, 창고명, 창고 | storage_center → dest_warehouse |
| 기준일자, 재고일자 | stock_date |
| 재고원가, 원가 | unit_cost |
| 재고금액 | total_price (없으면 0) |

---

## 3. 센터 필드 분리

- **inbound_center** (입고)
- **outbound_center** (출고)
- **storage_center** (재고)

DB 저장 시 각각 `dest_warehouse`로 매핑.

---

## 4. warehouse_group 분류

| 센터명 | warehouse_group |
|--------|-----------------|
| 테이칼튼, 테이칼튼 1공장 | 쿠팡 |
| 제이에스, 컬리, 기타 | 일반 |

---

## 5. 사용법

### 로컬 (Python)
```bash
python scripts/integrated_sync.py "경로/엑셀.xlsx"
python scripts/integrated_sync.py "경로/엑셀.xlsx" --debug
```

### 웹
- `ProductionSheetUploader` 컴포넌트에서 `parseProductionSheet(file)` 호출
- API: `POST /api/production-sheet-upload` 에 JSON 전송

---

## 6. 검증

동일한 엑셀 파일을:
1. 로컬: `python scripts/integrated_sync.py "파일.xlsx" --dry-run`
2. 웹: 업로드 후 API 응답 확인

**동일한 데이터 구조와 값**이 생성되어야 합니다.

---

## 7. 주의사항

- **헤더 2행 고정**: 자동 탐색 없음
- **로컬 전용/웹 전용 규칙 금지**: 모든 규칙은 common / excelParser에만 존재
- **center 필드 통합 금지**: inbound_center, outbound_center, storage_center 분리 유지
