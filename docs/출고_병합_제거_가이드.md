# 출고 병합 로직 제거 가이드

## 변경 요약

- **이전**: (product_code, outbound_date, dest_warehouse) 기준 병합 → 2965건 → 2798건
- **이후**: 1 row = 1 출고 트랜잭션, 병합 없음 → 2965건 그대로 적재

## 1. 수정한 파일

| 파일 | 변경 내용 |
|------|-----------|
| src/lib/commitProductionSheet.ts | outbound 병합 제거, 각 행 그대로 insert |
| src/app/api/production-sheet-validate/route.ts | outboundByKey 병합 제거, outboundCount = outbound.length |
| src/lib/previewTokenStore.ts | validation 타입에서 병합 관련 필드 제거 |
| src/components/ProductionSheetUploader.tsx | 출고 건수 원본만 표시, 병합 UI 제거 |
| scripts/migrate_outbound_no_merge.sql | unique index 제거 마이그레이션 |

## 2. DB 스키마

- **PK**: `id` (UUID) - 각 행 고유
- **unique 제약 제거**: (product_code, outbound_date, dest_warehouse) - 동일 키 여러 행 허용
- **마이그레이션**: `scripts/migrate_outbound_no_merge.sql` Supabase SQL Editor에서 실행

## 3. 기존 데이터 폐기 여부

| 상황 | 권장 |
|------|------|
| 기존 2798건 (병합됨) | **재업로드 권장** - 동일 엑셀 재업로드 시 당월 outbound_date 삭제 후 2965건 적재 |
| 마이그레이션만 실행 | unique 제약만 제거, 기존 2798건 유지. 다음 업로드부터 1 row = 1 트랜잭션 |

**권장**: 마이그레이션 실행 후, 생산수불현황 엑셀을 다시 업로드하여 2965건으로 재적재.

## 4. 집계 (KPI, 그래프, 소진예측)

- inventory_outbound를 `SUM(quantity)` 등으로 집계
- 행 수가 2965건으로 늘어나도 집계 결과(총 수량)는 동일
- 별도 코드 수정 불필요
