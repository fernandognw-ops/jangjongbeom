## 2. 재배포 여부

- **환경변수**: 변경 없음 → 재배포 불필요
- **코드**: quick API에 `_supabase_project_ref` 검증 필드 추가 → **재배포 완료**
- **NEXT_PUBLIC_* 참고**: 값 변경 시에는 **반드시 재배포** 필요 (빌드 시점에 번들에 포함)

---

## 3. inventory_sync 영향 여부

| 항목 | 결과 |
|------|------|
| 테이블 행 수 | 2건 (B9T3EQSMNHVK, TQKAXFDNHHQK) |
| 화면 복원 영향 | **조건부** – `refresh()` 실패 또는 "로컬 모드로 전환" 시, `getDefaultWorkspaceId()` 또는 `getStoredSyncCode()`가 있으면 inventory_sync에서 JSON 백업을 가져와 localStorage에 복원 |
| 정리 필요 | Supabase를 단일 출처로 사용할 경우, 불필요한 sync_code 행 삭제 권장 |

**확인**: `npm run check-inventory-sync`

---

## 4. localStorage 영향 여부

| 조건 | 영향 |
|------|------|
| useSupabaseInventory = **true** (Supabase 사용) | **없음** |
| useSupabaseInventory = **false** (로컬 모드) | **있음** – baseStock, transactions, products 등 표시 |

**삭제 방법**: `docs/localStorage_정리_가이드.md` 참고
