# 웹 업로드 구현 결과 보고

## 1. 업로드 UI 위치

| 상태 | 위치 |
|------|------|
| **데이터 없음** (empty_data) | 페이지 최상단 (배너 바로 아래) |
| **데이터 있음** (ok) | KPI 카드 아래 |

- `ProductionSheetUploader`는 `useSupabaseInventory`일 때 항상 렌더링됨
- 데이터 없을 때는 가장 먼저 보이도록 상단 배치

---

## 2. validate 호출 결과

**엔드포인트**: `POST /api/production-sheet-validate`

**요청**: FormData (file: Excel)

**응답**:
- `ok`, `validation`, `previewToken`
- validation: rawdataCount, inboundCount, outboundCount, stockCount, totalStockValue, destWarehouseDistribution, snapshotDates, destWarehouseValid

**화면 표시**:
- rawdata 건수
- 입고 건수
- 출고 건수
- 재고 건수
- 재고 총 금액
- 일반 / 쿠팡 분포
- snapshot_date

---

## 3. commit 호출 결과

**엔드포인트**: `POST /api/production-sheet-commit`

**요청**: `{ previewToken }`, 헤더 `x-source: web`

**성공 시**:
- 성공 메시지 표시 (입고 N건, 출고 N건, 재고 N건)
- 1.5초 후 `refresh()` 호출 → `/api/inventory/quick` 재조회
- KPI 갱신

---

## 4. 업로드 후 KPI 변화

- `refresh()`가 `/api/inventory/quick` 재호출
- `productCount`, `totalValue`, `totalQuantity`, `totalSku` 갱신
- 대시보드 KPI 카드 자동 반영

## 6. 환경 확인

- `ALLOW_DB_WRITE`가 `"false"`이면 commit API 503 반환
- `.env.local`에 미설정 또는 `true`이면 정상 동작
