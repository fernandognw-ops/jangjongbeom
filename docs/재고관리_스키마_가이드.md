# 재고 관리 시스템 - 스키마 및 기능 가이드

`0310_생산수불현황` Excel 구조 기반으로 설계된 DB 스키마와 계산 로직입니다.

---

## 1. DB 스키마 (PostgreSQL / SQLite)

### 파일 위치
- **PostgreSQL (Supabase)**: `supabase-schema-inventory.sql`
- **SQLite**: `schema-sqlite.sql`

### 주요 테이블

| 테이블 | 설명 |
|--------|------|
| `products` | 제품 마스터 (품목코드, 품목구분, 원가, 매출구분) |
| `inbound` | 입고 내역 (매출구분: 쿠팡/일반) |
| `outbound` | 출고 내역 (매출구분: 쿠팡/일반) |
| `stock_snapshot` | 일별 재고 스냅샷 (시계열 분석용) |
| `logistics_cutoff_config` | 물류센터 마감 시각 설정 |

### 매출 구분 (sales_channel)
- **coupang**: 쿠팡 물류
- **general**: 일반 (제이에스, CJ 등)

### Supabase 적용 방법
1. Supabase 대시보드 → SQL Editor
2. `supabase-schema-inventory.sql` 내용 붙여넣기
3. Run 실행

---

## 2. 실시간 가용 재고 (물류센터 마감 시차 보정)

### 로직
- **쿠팡**: 18:00 마감 → 18시 이전에는 당일 쿠팡 출고분이 아직 반영되지 않음
- **일반**: 23:59 마감 → 당일 일반 출고분 마감 전까지 미반영

### 사용
```typescript
import { computeRealTimeAvailableStock, getTodayOutboundByItem } from "@/lib/inventoryCalculations";

const todayOut = getTodayOutboundByItem(transactions);
const available = computeRealTimeAvailableStock(
  currentStock,
  todayOut[itemId] ?? { coupang: 0, general: 0 },
  new Date()
);
```

## 3. 재고 소진일(Run-out Date) 예측

### 기준
- 최근 30일 출고 평균 (실제 거래일 기준)
- 1년+ 데이터 보유 시 `lookbackDays: 90` 권장

### 대시보드
- 메인 페이지에 **재고 소진일 예측** 카드 표시
- 7일 이내 소진 예상 품목은 빨간색 강조

---

## 4. Excel 업로드 시 매출구분 반영

입고/출고 CSV·Excel에 **매출구분** 컬럼이 있으면 자동 파싱:
- `쿠팡` → coupang
- 그 외 (일반, 제이에스 등) → general
