## 4. 웹 업로드 2단계 구조

### 1단계: 파일 업로드 → 파싱
- Excel 파일 드래그/선택
- 서버 `/api/production-sheet-upload/parse` 에서 파싱
- 검증 결과 표시

### 2단계: 검증 확인 → DB 반영
- 검증 항목: 품목 수, 입고/출고/재고 건수, 재고 총 금액, dest_warehouse 분포, snapshot_date
- dest_warehouse 오류 시 반영 버튼 비활성화
- "DB 반영" 클릭 → `/api/production-sheet-upload` (action=apply)

---

## 5. 로컬 integrated_sync.py

- **기본값**: dry-run (DB 반영 없음)
- **--validate**: 웹 vs Python 파싱 비교
- **--apply**: 실제 DB 반영 (비권장, 웹 업로드 사용 권장)

```bash
# 파싱/검증만 (기본)
python scripts/integrated_sync.py "경로/수불현황.xlsx"

# DB 반영 (비권장)
python scripts/integrated_sync.py "경로/수불현황.xlsx" --apply
```

---

## 6. 적재 규칙

- dest_warehouse: 반드시 "일반" / "쿠팡"
- 재고: snapshot 기반 (누적 계산 금지)
- 당월 데이터만 업데이트
- PK: (product_code, dest_warehouse, snapshot_date)
