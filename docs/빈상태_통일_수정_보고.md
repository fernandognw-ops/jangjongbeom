# 빈 상태 통일 수정 보고

## 1. 섹션별 데이터 소스 표

| 섹션 | API/테이블 | 상태값 | 비고 |
|------|------------|--------|------|
| 상단 배너 | - | supabaseFetchStatus, supabaseFetchError | idle/ok 시 숨김, fetch_error/empty_data 시 표시 |
| 업로드 UI 노출 | - | supabaseFetchStatus === "empty_data" && useSupabaseInventory | DB 0건 시 상단 배치 |
| 재고 금액 KPI | /api/inventory/quick 또는 /api/inventory/snapshot | kpiData.totalValue | snapshot 단일 출처 |
| 재고 소진예측(RunOutDateCard) | - | stock, transactions (context) | supabaseDerived 또는 localDerived |
| 기타 대시보드 카드 | DashboardBoxHero, TopSkuByCategoryDashboard | stockSnapshot, inventoryProducts, kpiData | snapshot 기반 |

## 2. 실제 호출 API

| API | 호출 시점 | 용도 |
|-----|------------|------|
| /api/inventory/quick | refresh() 1단계 (5초 타임아웃) | 초고속 KPI/스냅샷 (inventory_stock_snapshot) |
| /api/inventory/snapshot | quick 실패 시 fallback, 또는 2단계 전체 | 스냅샷 + dailyVelocity, stockByChannel |
| /api/inventory/summary | refresh() 내 (5초 race) | 제품 메타 (category 등) |
| /api/category-trend | refresh() 3단계 백그라운드 | 판매·입고 추세 |
| /api/forecast | refresh() 3단계 백그라운드 | AI 수요 예측 |
| /api/inventory/data | (페이지에서 직접 호출 없음) | - |
| /api/dashboard | (페이지에서 직접 호출 없음) | - |

## 3. 잘못된 분기 원인

1. **no_snapshot을 fetch_error로 처리**: quick/snapshot API가 `error: "no_snapshot"`을 반환할 때 `if (data.error) throw`로 예외 처리 → catch에서 `fetch_error` 설정 → `useSupabaseInventory=false` → localStorage 복원
2. **localStorage 분기로 전환**: fetch_error 시 localStorage에서 이전 데이터 복원 → KPI/소진예측에 localStorage 값 표시
3. **업로드 UI 숨김**: 업로드 UI는 `useSupabaseInventory && supabaseFetchStatus === "empty_data"`일 때만 상단 노출. fetch_error 시 useSupabaseInventory=false라 Supabase 분기가 아니어서 업로드 UI 미표시

## 5. 수정 후 빈 상태에서의 기대 동작

- **배너**: "Supabase 연결됨, 데이터 없음" (empty_data)
- **업로드 UI**: 상단에 표시
- **KPI**: 0원, 0건, 0EA, 0박스
- **소진예측 카드**: "재고·출고 데이터가 없습니다. 위 [생산수불현황 업로드]에서..." 메시지
- **이전/다른 데이터**: 표시되지 않음 (localStorage 미복원)
