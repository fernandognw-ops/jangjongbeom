## 2) DB 복구 (잘못 적재된 데이터 삭제)

### 삭제 기준

| 테이블 | 식별 컬럼 | 삭제 조건 |
|--------|-----------|-----------|
| inventory_inbound | created_at | >= 2026-03-18 07:00:00+09 |
| inventory_outbound | created_at | >= 2026-03-18 07:00:00+09 |
| inventory_stock_snapshot | updated_at | >= 2026-03-18 07:00:00+09 |

sync가 **당월만** 삭제 후 upsert하므로, 3월 데이터 전체가 잘못되었을 수 있습니다.  
이 경우 `02b_delete_march_only.sql` 사용 (당월 inbound/outbound 전체 삭제).

### 실행 순서

1. **Supabase 대시보드** → SQL Editor
2. `scripts/recovery_0318/01_backup_before_delete.sql` 실행  
   → row count 확인
3. `scripts/recovery_0318/02_delete_bad_data.sql` 실행  
   (또는 3월 전체 삭제 시 `02b_delete_march_only.sql`)
