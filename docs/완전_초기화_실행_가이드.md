# 완전 초기화 실행 가이드

웹 업로드 단일 반영 전, 기존 데이터 전체 제거용

---

## 1. 삭제 전 row count 확인

Supabase SQL Editor에서 실행:

```sql
SELECT 'inventory_sync' AS tbl, COUNT(*) AS cnt FROM inventory_sync
UNION ALL SELECT 'inventory_inbound', COUNT(*) FROM inventory_inbound
UNION ALL SELECT 'inventory_outbound', COUNT(*) FROM inventory_outbound
UNION ALL SELECT 'inventory_stock_snapshot', COUNT(*) FROM inventory_stock_snapshot
UNION ALL SELECT 'inventory_products', COUNT(*) FROM inventory_products;
```

**결과 기록**: 삭제 전 row count

---

## 2. (선택) 백업

필요 시 아래 실행:

```sql
CREATE TABLE inventory_products_backup_reset AS SELECT * FROM inventory_products;
CREATE TABLE inventory_inbound_backup_reset AS SELECT * FROM inventory_inbound;
CREATE TABLE inventory_outbound_backup_reset AS SELECT * FROM inventory_outbound;
CREATE TABLE inventory_stock_snapshot_backup_reset AS SELECT * FROM inventory_stock_snapshot;
CREATE TABLE inventory_sync_backup_reset AS SELECT * FROM inventory_sync;
```

---

## 3. 전체 삭제

**Supabase SQL Editor에서 바로 실행**: `scripts/supabase_reset_run.sql` 전체 복사 후 Run

또는 `scripts/full_reset_inventory.sql` 전체 실행  
또는 아래만 실행:

```sql
TRUNCATE TABLE inventory_sync;
TRUNCATE TABLE inventory_inbound CASCADE;
TRUNCATE TABLE inventory_outbound CASCADE;
TRUNCATE TABLE inventory_stock_snapshot CASCADE;
TRUNCATE TABLE inventory_products CASCADE;
```

---

## 4. 삭제 후 row count = 0 확인

동일한 count 쿼리 재실행 → 모든 테이블 cnt = 0

---

## 5. API 검증

개발 서버 실행 후:

```bash
npm run dev
# 다른 터미널에서
node scripts/verify_reset_complete.mjs
```

또는 Vercel 배포 URL:

```bash
node scripts/verify_reset_complete.mjs https://your-app.vercel.app
```

---

## 6. 웹 화면 확인

1. 브라우저에서 대시보드 접속
2. **새로고침** (또는 Ctrl+Shift+R 강력 새로고침)
3. 확인:
   - KPI: 0건, 0원, 0EA, 0박스
   - 그래프: "최근 12개월 출고 데이터가 없습니다"
   - 재고 테이블: 빈 상태
   - 배너: "Supabase 연결됨, 데이터 없음"

---

## 7. localStorage / 캐시 정리

- `empty_data` 시 앱이 자동으로 `inventory-*` 키 삭제
- 수동 정리: 개발자 도구 → Application → Local Storage → `inventory-` 로 시작하는 키 삭제
