# 재고 스냅샷 집계 수정 가이드

## 수정 내용 요약

### 1. 월별 재고 집계 (category-trend)

**기존**: 최신 snapshot만 전체 월에 사용 → 과거 월 값이 모두 동일

**수정**: 해당 월의 마지막 snapshot_date만 사용
- 각 월별로 `max(snapshot_date)` 조회
- 해당 날짜의 행만 집계

### 2. 채널 집계 통일

**기존**: 상위 총합이 dest_warehouse 무시하고 "일반"으로만 집계

**수정**: 모든 API에서 `normalizeDestWarehouse()` 사용
- "일반" | "쿠팡" 만 허용
- 총합 = SUM(일반) + SUM(쿠팡)

### 3. 적재 규칙 (production-sheet-upload)

- PK: (product_code, dest_warehouse, snapshot_date)
- 당월 날짜만 replace
- 과거 월 데이터 절대 변경 금지

### 4. DB 검증 SQL

```sql
-- snapshot_date별 건수 확인
SELECT snapshot_date, COUNT(*) AS row_count
FROM inventory_stock_snapshot
GROUP BY snapshot_date
ORDER BY snapshot_date DESC;
```

### 5. 검증 기준

1. **월별 그래프**: 값이 월마다 다르게 표시
2. **채널 분포**: 총합 = 쿠팡 + 일반
3. **당월**: 최신 snapshot_date 반영
4. **과거 데이터**: 변경되지 않음
