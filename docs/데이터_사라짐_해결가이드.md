# 데이터가 사라지는 문제 해결 가이드

## 원인 1: Supabase 프로젝트 불일치 (가장 흔함)

**증상**: bulk-upload는 성공하는데 대시보드에 0원·0건만 표시됨

**원인**: bulk-upload는 **localhost API**를 호출하고, 대시보드는 **배포 URL**(vercel.app 등)에서 열림.  
로컬 `.env.local`의 Supabase와 Vercel 환경변수의 Supabase가 **서로 다른 프로젝트**일 수 있음.

**해결**:
1. **로컬에서만 사용할 때**: `http://localhost:3000` 으로 접속 후 bulk-upload 실행
2. **배포 URL에서 사용할 때**: Vercel 대시보드 → 프로젝트 → Settings → Environment Variables 에서  
   `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`가 `.env.local`과 **동일한 Supabase 프로젝트**를 가리키는지 확인

## 원인 2: bulk-upload 실행 시 dev 서버 미실행

**증상**: bulk-upload 실행 시 "fetch failed" 또는 HTML 응답

**해결**: `npm run dev` 실행 후 bulk-upload 실행

---

## 원인 3: reset_march_2026_data.sql 실행

`scripts/reset_march_2026_data.sql`는 2026-03-01 이후 입고/출고를 삭제합니다.  
실수로 실행했다면 bulk-upload 다시 실행.

---

## 원인 4: RLS 정책

Supabase Table Editor에서 RLS가 활성화되어 있으면, anon key로 읽기/쓰기가 차단될 수 있습니다.

**확인**: `supabase-schema-inventory-alt.sql`, `supabase-schema-stock-snapshot.sql`에 있는  
`USING (true) WITH CHECK (true)` 정책이 적용되어 있는지 확인

---

## 진단 방법

1. **데이터 진단 버튼**: 대시보드에서 "데이터 진단" 클릭 → Supabase 프로젝트 ID, row 수 확인
2. **bulk-upload 검증**: bulk-upload 실행 후 `[5] 검증 중...` 결과 확인
   - DB 실제 row 수가 0이면 → Supabase 프로젝트 불일치 가능성

---

## 권장 순서

1. `npm run dev` 실행
2. `npm run bulk-upload "폴더경로"` 실행
3. **http://localhost:3000** 으로 접속 (배포 URL 아님)
4. 새로고침 버튼 클릭
