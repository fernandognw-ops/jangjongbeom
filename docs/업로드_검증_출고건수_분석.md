# 업로드 검증 출고 건수·채널 분포 분석

## 1. 출고 건수 단계별 흐름

| 단계 | 설명 | 예시 |
|------|------|------|
| 원본 | 출고 시트 데이터 행 수 (HEADER_ROW+1 이후) | 2965건 |
| 필터 후 | validProductCode, qty>0, dateStr 통과한 행 | 2965건 (0건 제외 시) |
| DB 적재 | (product_code, outbound_date, dest_warehouse) 기준 병합 후 | 2798건 |
| 병합 | 동일 키로 합쳐진 행 수 | 167건 |

**167건 차이 원인**: 동일 품목·날짜·창고 조합이 여러 행에 있으면 수량을 합산하고 1건으로 적재함.

## 2. 제외/병합 규칙

### 필터 제외 (원본 → 필터 후)
- `validProductCode(code)` 실패: 품번 5자 미만, 숫자 비율 50% 미만, "합계"/"소계" 포함
- `qty <= 0`: 수량 0 이하
- `!dateStr`: 날짜 파싱 실패

### 병합 (필터 후 → DB 적재)
- 키: `product_code|outbound_date|dest_warehouse`
- 동일 키 행: 수량 합산 후 1건으로 적재

## 3. 일반/쿠팡 분포 의미

| 라벨 | 의미 |
|------|------|
| 입고+출고+재고 합산 행 기준 채널 분포 | 입고·출고·재고 시트의 모든 행을 dest_warehouse로 분류한 건수 |
| 채널 분포 상세 | 입고/출고/재고 각각의 일반·쿠팡 건수 |

- **일반 2413 / 쿠팡 1138**: 입고+출고+재고 전체 행 수 기준 (3541건 합계)
