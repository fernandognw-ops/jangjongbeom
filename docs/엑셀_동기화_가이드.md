# 엑셀 → Supabase 통합 동기화

## 사용법

**파일만 올리면 테이블 업데이트가 됩니다.**

```bash
npm run sync-excel "경로/엑셀.xlsx"
```

예: `26년 0311_생산수불현황.xlsx` 파일을 같은 위치에 두고 실행하면 됩니다.

## 필수 조건

- 엑셀 파일에 **4개 시트**: rawdata, 입고, 재고, 출고
- `.env.local`에 Supabase URL, KEY 설정
- `pip install pandas openpyxl supabase`

## 옵션

| 명령 | 설명 |
|------|------|
| `npm run sync-excel "파일.xlsx"` | Upsert (테이블 비우지 않음, 재고만 최신으로 교체) |
| `npm run sync-excel "파일.xlsx" -- --reset` | 기존 데이터 삭제 후 전체 재업로드 |
| `npm run sync-excel "파일.xlsx" -- --dry-run` | DB 반영 없이 시뮬레이션 |

## 시트 → 테이블 매핑

| 엑셀 시트 | Supabase 테이블 | Upsert 키 |
|-----------|-----------------|-----------|
| rawdata | inventory_products | product_code |
| 입고 | inventory_inbound | product_code + inbound_date |
| 재고 | inventory_stock_snapshot | (기존 삭제 후 전체 교체) |
| 출고 | inventory_outbound | product_code + outbound_date + sales_channel |

## 컬럼 매핑 (정확 일치 우선)

- **rawdata**: 품목코드, 품목명→제품명, 제품원가표(개당), 품목, 입수량
- **입고**: 품목코드, 품목명→제품명, **품목구분**→category, 입수량, **수량**(또는 품목수), 입고처, 입고일자(또는 입고일자 주차), 원가, 합계원가
  - 총 입고 = 수량 열 합계
  - 카테고리별 = 품목구분별 수량 합계
- **재고**: 품목코드, 품목명→제품명, 품목구분, 입수량, 수량, 창고명, 재고일자, 원가, 재고원가
- **출고**: 품목코드, 품목명→제품명, 품목/품목구분, 입수량, 수량, 출고처, 출고일자, 원가, 합계, 매출구분

## 원가 보완 (unit_cost / unit_price / total_price)

엑셀에 원가·합계 컬럼이 없거나 0인 경우, `inventory_products`의 `unit_cost`로 자동 보완합니다.

- **재고**: unit_cost, total_price (= quantity × unit_cost)
- **입고**: unit_price, total_price
- **출고**: unit_price, total_price

기존 DB 데이터만 수정할 때: `npm run update-stock-costs`

## 품목코드 → 품목구분(카테고리) 매칭

필터(마스크, 생활용품 등)에 제품이 올바르게 표시되려면 `product_code` → `category`(품목구분) 매핑이 필요합니다.

- **재고** 시트 품목구분 → `inventory_stock_snapshot.category` **(우선 적용)**
- **rawdata** 시트 품목 → `inventory_products.group_name`

**우선순위**: 재고 시트 품목구분 > rawdata 품목 > 입출고 category > "생활용품"

- 엑셀 재고 시트에서 품목구분 수정 후: `npm run update-category "경로/엑셀.xlsx"` 실행
- 특정 품목 수동 수정: `scripts/fix_product_category.sql` 실행 (product_code, category 값 수정)
- 매칭 보완: `scripts/sync_snapshot_category_from_products.sql` 실행

## 대시보드 연동

- **inventory_current_products**: sync-excel 실행 시 자동 동기화 (재고+입고+출고 품목코드)
- 테이블 없으면: `scripts/create_current_products_table.sql` 실행 후 `npm run sync-current-products`
- 입고/출고 조회: 최근 6개월 누적 데이터 표시
