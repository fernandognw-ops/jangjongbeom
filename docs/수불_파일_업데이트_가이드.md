# 수불 파일 업데이트 가이드

**생산수불현황 동기화는 대시보드 업로드 UI에서만 진행합니다.**

담당자가 매일 아침 집계한 `생산수불현황.xlsx`를 드래그 앤 드롭하여 DB를 갱신합니다. 파일이 업로드되면 자동으로 동기화됩니다.

---

## 1. 업로드 위치

대시보드 메인 페이지의 **생산수불현황 업로드** 영역을 사용합니다.

- 파일을 드래그 앤 드롭하거나 클릭하여 선택
- 업로드 즉시 입고·출고·재고 데이터가 DB에 반영됨
- 중복 데이터는 자동으로 병합됨

---

## 2. 파일 요구사항

### 필수 시트

엑셀에 아래 3개 시트가 있어야 합니다.

| 시트 | 용도 |
|------|------|
| **입고** | 입고 내역 |
| **출고** | 출고 내역 |
| **재고** | 현재 재고 수량·원가 |

### 파일 형식

- `.xlsx` 또는 `.xls` (Excel)
- 파일명 예: `0316_생산수불현황.xlsx`, `26년 0320_생산수불현황.xlsx`

---

## 3. 업로드 절차

1. 담당자가 집계한 `생산수불현황.xlsx` 파일 준비
2. 대시보드 접속 → **생산수불현황 업로드** 영역으로 이동
3. 파일을 드래그 앤 드롭 또는 클릭하여 선택
4. 파싱·저장 완료 메시지 확인
5. 대시보드가 자동으로 새로고침됨

---

## 4. 동기화 결과

업로드 시 다음 테이블이 갱신됩니다.

| 테이블 | 내용 |
|--------|------|
| `inventory_inbound` | 입고 내역 (product_code + inbound_date 기준 병합) |
| `inventory_outbound` | 출고 내역 (product_code + outbound_date + sales_channel 기준 병합) |
| `inventory_stock_snapshot` | 재고 스냅샷 (product_code 기준 덮어쓰기) |
| `inventory_current_products` | 현재 운영 품목 목록 |
| `inventory_products` | 신규 품목 자동 등록 (없는 경우) |

---

## 5. 주의사항

- **이 업로드 UI가 유일한 수불 동기화 경로입니다.** 로컬 폴더나 `npm run sync-excel`은 사용하지 않습니다.
- `snapshot_date`는 업로드 실행일로 저장됩니다.
- 입고·출고는 누적 추가(INSERT)이며, 재고는 최신 값으로 덮어씁니다.

---

## 6. 트러블슈팅

### "필수 시트가 없습니다" 오류

- 입고·출고·재고 시트가 모두 있는지 확인하세요.
- 시트 이름에 공백·오타가 없는지 확인하세요.

### "유효한 데이터를 찾을 수 없습니다"

- 각 시트에 품목코드, 수량, 일자 열이 있는지 확인하세요.
- 헤더 행(보통 3행) 아래부터 데이터가 있는지 확인하세요.

### 수량이 예상과 다름

- 재고 시트의 **수량** 또는 **재고수량** 컬럼이 올바르게 인식되는지 확인하세요.
- 품목코드 형식(숫자 비율 등)이 올바른지 확인하세요.
