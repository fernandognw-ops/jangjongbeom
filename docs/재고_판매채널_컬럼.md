# 재고 시트: 판매 채널 컬럼

## 정책

| 엑셀 컬럼 | DB 컬럼 | 설명 |
|-----------|---------|------|
| **판매 채널** (J열 등, 헤더 동의어: `판매채널`, `판매 채널명`) | `inventory_stock_snapshot.sales_channel` | `normalizeSalesChannelKr` → `"쿠팡"` \| `"일반"` |
| **보관 센터** / 창고명 등 | `inventory_stock_snapshot.dest_warehouse` | 물리 창고·센터명 (빈 값 → `미지정`) |

- 보관센터 문자열로 쿠팡/일반을 **추론하지 않음** (레거시 `toDestWarehouse`는 입고 등에만 사용).
- DB 적재·대시보드 `channelTotals`는 **`sales_channel`** 기준.

## 마이그레이션

기존 DB는 `scripts/migrate_snapshot_sales_channel.sql` 실행 후, 생산수불 엑셀을 한 번 업로드해 `dest_warehouse`에 실제 보관센터명이 채워지도록 하는 것을 권장합니다.

## 검증

```bash
npx tsx scripts/sum_stock_snapshot_from_excel.ts <엑셀경로>
npx tsx scripts/verify_quick_channel_totals.ts
```

SQL:

```sql
SELECT sales_channel, SUM(quantity) FROM inventory_stock_snapshot
WHERE snapshot_date = (SELECT MAX(snapshot_date) FROM inventory_stock_snapshot)
GROUP BY 1;
```
