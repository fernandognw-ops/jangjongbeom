# 시스템 정상화 가이드

로딩 화면에 멈추거나 데이터가 안 보일 때 아래 순서로 진행하세요.

---

## 1. 서버 재시작 (권장)

```powershell
cd c:\Users\pc\inventory-system
powershell -ExecutionPolicy Bypass -File .\scripts\start-server.ps1
```

- 포트 3007 사용 중인 프로세스 자동 종료 후 프로덕션 서버 시작
- **http://localhost:3007** 접속

---

## 2. Supabase 데이터 동기화

```powershell
cd c:\Users\pc\inventory-system
python scripts/integrated_sync.py "C:\Users\pc\Desktop\장종범\인수 인계서\물류 재고 관리 시스템 구축\수불 마감 자료" --reset
```

- `--reset`: 기존 데이터 삭제 후 엑셀 기준으로 재적재
- 폴더 경로 지정 시 해당 폴더에서 최신 수불현황 엑셀 자동 검색

---

## 3. 수동으로 포트 정리 후 서버 시작

```powershell
# 포트 3007 프로세스 확인
netstat -ano | findstr ":3007"

# PID 확인 후 종료 (예: PID가 12345인 경우)
Stop-Process -Id 12345 -Force

# 3초 대기 후 서버 시작
cd c:\Users\pc\inventory-system
npm run start
```

---

## 4. 로컬 모드로 전환

Supabase 연결이 안 될 때:

- 로딩 화면에서 **"로컬 모드로 전환"** 버튼 클릭
- localStorage 기반으로 대시보드 사용 가능

---

## 요약

| 증상 | 해결 |
|------|------|
| 로딩 화면에 멈춤 | start-server.ps1 실행 또는 새로고침 버튼 |
| 데이터 0원/0건 | integrated_sync.py --reset 실행 |
| 포트 사용 중 | start-server.ps1 (자동 종료 후 시작) |
