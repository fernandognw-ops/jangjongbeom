# 완전 초기화 복구 가이드 (0건 상태)

첫 테스트 업로드 전, DB를 0건 상태로 초기화하는 절차입니다.

---

## 0. 사전 준비 (최초 1회)

`inventory_upload_logs` 테이블이 없으면 `scripts/create_inventory_upload_logs.sql` 실행.

---

## 1. 초기화 SQL (Supabase SQL Editor)

`scripts/supabase_reset_run.sql` 또는 `scripts/full_reset_inventory.sql` 사용.

### 대상 테이블 (TRUNCATE 순서)

| 순서 | 테이블 | 설명 |
|------|--------|------|
| 1 | inventory_sync | 연동 데이터 (독립) |
| 2 | inventory_inbound | 입고 |
| 3 | inventory_outbound | 출고 |
| 4 | inventory_stock_snapshot | 재고 스냅샷 |
| 5 | inventory_products | 제품 마스터 |

### inventory_upload_logs

- **유지** (TRUNCATE하지 않음)
- 업로드 이력용 테이블로, 초기화 대상 아님
- 첫 테스트 업로드 후 1건 추가됨

### inventory_current_products (선택)

- 테이블이 있으면 `TRUNCATE TABLE inventory_current_products;` 추가 가능
- 없으면 무시

---

## 2. 실행할 SQL (복사용)

```sql
-- [1] 삭제 전 row count
SELECT '삭제 전' AS phase, 'inventory_sync' AS tbl, COUNT(*) AS cnt FROM inventory_sync
UNION ALL SELECT '삭제 전', 'inventory_inbound', COUNT(*) FROM inventory_inbound
UNION ALL SELECT '삭제 전', 'inventory_outbound', COUNT(*) FROM inventory_outbound
UNION ALL SELECT '삭제 전', 'inventory_stock_snapshot', COUNT(*) FROM inventory_stock_snapshot
UNION ALL SELECT '삭제 전', 'inventory_products', COUNT(*) FROM inventory_products;

-- [2] TRUNCATE
TRUNCATE TABLE inventory_sync;
TRUNCATE TABLE inventory_inbound CASCADE;
TRUNCATE TABLE inventory_outbound CASCADE;
TRUNCATE TABLE inventory_stock_snapshot CASCADE;
TRUNCATE TABLE inventory_products CASCADE;

-- [3] 삭제 후 row count (모두 0이어야 함)
SELECT '삭제 후' AS phase, 'inventory_sync' AS tbl, COUNT(*) AS cnt FROM inventory_sync
UNION ALL SELECT '삭제 후', 'inventory_inbound', COUNT(*) FROM inventory_inbound
UNION ALL SELECT '삭제 후', 'inventory_outbound', COUNT(*) FROM inventory_outbound
UNION ALL SELECT '삭제 후', 'inventory_stock_snapshot', COUNT(*) FROM inventory_stock_snapshot
UNION ALL SELECT '삭제 후', 'inventory_products', COUNT(*) FROM inventory_products;
```

## 3. 초기화 후 검증

### /api/inventory/quick 기대 결과

| 항목 | 기대값 |
|------|--------|
| productCount | 0 |
| totalValue | 0 |
| totalQuantity | 0 |
| totalSku | 0 |
| error | no_snapshot |

### 검증 명령

```bash
npm run verify-reset -- http://localhost:3007
```

또는:

```bash
curl -s http://localhost:3007/api/inventory/quick
```

브라우저: `http://localhost:3007/api/inventory/quick`

**성공 시**: `✅ 초기화 정상 완료. (no_snapshot + 모든 값 0)`

---

## 4. 0건 확인 후 → 웹 업로드 테스트 (3~6단계)

0건 상태가 확인되면 아래 순서로 진행:

| 단계 | 내용 |
|------|------|
| 3 | `http://localhost:3007` → 생산수불현황 Excel 드래그/선택 |
| 4 | 업로드 전 검증 수치 확인 (rawdata, inbound, outbound, stock, total value, 일반/쿠팡, snapshot_date) |
| 5 | 수치 이상 없으면 **DB 반영** 클릭 |
| 6 | `scripts/verify_after_first_upload.sql` 실행 → 결과 기록 |
